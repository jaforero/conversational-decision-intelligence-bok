# Sistema de marca del CDI-BoK

El CDI-BoK utiliza una arquitectura de **marca endosada**:

`Javier Forero → CDI-BoK → PULSE / Dashboards Practice Lab`

La marca personal respalda autoría y coherencia visual, pero no sustituye la identidad institucional del Body of Knowledge ni convierte páginas normativas en piezas promocionales.

## Contratos de Sprint 0

- `brand-source.yml`: procedencia, autoridad y versión de la guía.
- `tokens.json`: fuente de verdad tecnológica para valores de diseño.
- `tokens.css`: distribución web de los tokens aprobados.
- `asset-manifest.yml`: estado, licencia, procedencia y reglas de uso de cada activo.

No existe autorización en esta release para inventar un logotipo CDI-BoK, copiar una tipografía o declarar conformidad visual sin evidencia.
