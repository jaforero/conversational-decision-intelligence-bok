# From a priority to a decision that learns

This path turns CDI-BoK principles into an applicable journey. You do not need to start with AI, a dashboard or a perfect case. You need a **material priority**, a decision worth improving and willingness to record what you expect before acting.

## What you will build

The five modules provide inputs for a one-page **Decision Brief**:

1. a PDAMR decision frame;
2. an evidence and context contract;
3. a Decision Experience blueprint;
4. a Human–AI control matrix;
5. an outcome and learning contract.

This does not guarantee a good decision. It makes the decision more explicit, challengeable, governable and learnable.

## How to use each module

| Element | Purpose |
|---|---|
| **Guiding question** | Connect concepts to a decision |
| **Model** | Organize the problem with the fewest useful components |
| **Checklist** | Turn the model into an applicable review |
| **Stress test** | Expose assumptions, uncertainty and counterarguments |
| **Deliverable** | Produce a reusable input to the Decision Brief |

## Three entry paths

- **Sequential:** work from module 1 through 5 when building judgment from scratch.
- **By friction:** begin where the current chain breaks—decision, evidence, experience, control or learning.
- **Existing-solution review:** complete the [Decision Brief](decision-brief.md), then return to the first module with an undefended field.

!!! tip "Practical rule"
    If you cannot state the decision in one sentence, you need framing before more data. If you can state it but cannot say what evidence would change your choice, you need an evidence contract. If both are clear and nothing changes afterward, the gap is action, governance or learning.

## Outside this path

This path does not replace specialist training in statistics, causality, ethics, law, security, product design or change management. It does not turn an exercise into impact evidence. It reveals which additional discipline a specific decision requires.
