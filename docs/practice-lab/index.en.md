## A demo is not yet a case

Admission as a case requires at least decision and context; users and accountability; available data and limits; prior baseline; designed intervention; action taken; observed result; learning and PULSE relationship. ADR-018 defines `catalogued → selected → instrumented-not-executed → observed-noncausal → evaluated`. Publishing a demo cannot skip stages.

!!! warning "Evidence boundary"
    A compelling interface demonstrates design capability. It does not by itself prove better organizational decisions, higher-quality action or better outcomes.
