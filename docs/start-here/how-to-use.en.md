# How to use the CDI-BoK

The portal does not require linear reading. Choose a path based on the decision you need to strengthen.

## Recommended paths

| If you need to… | Start with… | Continue with… |
|---|---|---|
| Understand the discipline | Start here | CDI-BoK and Research |
| Design a decision practice | PULSE | Governance and Practice Lab |
| Evaluate a claim | Claims and evidence | Source registry |
| Implement in an organization | PULSE | Architectures and future cases |
| Contribute content | Governance | `CONTRIBUTING.md` on GitHub |

## Always read four signals

1. **Status:** idea, draft, review, candidate, approved, deprecated or superseded.
2. **Authority:** constitutional, normative, guidance, evidence or experiment.
3. **Version:** identifies the specific intellectual contract you are reading.
4. **Evidence:** states whether a claim is a definition, finding, hypothesis, aspiration or original proposal.

!!! example "Example"
    PULSE's official definition can be constitutional within the project while not constituting empirical evidence of generalizable effectiveness. Institutional authority and scientific strength are different dimensions.

## Prudence rule

Do not turn a demo into a case, a metaphor into a mechanism, an AI synthesis into a systematic review, or an aspiration into external recognition. The CDI-BoK makes these boundaries visible to improve decisions about its own knowledge.
