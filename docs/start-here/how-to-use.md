# Cómo usar el CDI-BoK

El portal no exige una lectura lineal. Elija una ruta según la decisión que necesita fortalecer.

## Rutas recomendadas

| Si necesita… | Comience por… | Continúe con… |
|---|---|---|
| Comprender la disciplina | Empieza aquí | CDI-BoK y Research |
| Diseñar una práctica decisional | PULSE | Governance y Practice Lab |
| Evaluar una afirmación | Claims y evidencia | Registro de fuentes |
| Implementar en una organización | PULSE | Arquitecturas y casos futuros |
| Contribuir contenido | Governance | `CONTRIBUTING.md` en GitHub |

## Lea siempre cuatro señales

1. **Estado:** idea, draft, review, candidate, approved, deprecated o superseded.
2. **Autoridad:** constitucional, normativa, guía, evidencia o experimento.
3. **Versión:** identifica el contrato intelectual concreto que está leyendo.
4. **Evidencia:** indica si una afirmación es definición, hallazgo, hipótesis, aspiración o propuesta original.

!!! example "Ejemplo"
    La definición oficial de PULSE puede ser constitucional dentro del proyecto y, al mismo tiempo, no constituir evidencia empírica de eficacia generalizable. Autoridad institucional y fuerza científica son dimensiones distintas.

## Regla de prudencia

No convierta una demo en caso, una metáfora en mecanismo, una síntesis de IA en revisión sistemática ni una aspiración en reconocimiento externo. El CDI-BoK hace visibles esas fronteras para mejorar la calidad de decisión sobre su propio conocimiento.
