# Release v0.8.0

`v0.8.0` establece una **línea base editorial y técnica estable** del portal CDI-BoK después de la ratificación de `v0.8.0-rc.1`. Integra el núcleo fundacional, la ruta de aprendizaje, el sistema de medición, el Practice Lab y los patrones conversacionales bajo navegación, trazabilidad y gates reproducibles.

## Qué se estabiliza

- el portal MkDocs y su arquitectura pública de 45 páginas;
- la navegación desde decisión y evidencia hasta acción, medición y aprendizaje;
- los contratos de build, auditoría semántica, regresión visual y accesibilidad automatizada;
- la identificación de autoridad, fuentes, claims, límites y estado de cada activo;
- la línea de versiones desde `v0.2.0` hasta el candidato fuente `v0.8.0-rc.1`;
- el mecanismo que crea el tag y GitHub Release solo desde el SHA validado de `main`.

## Autoridad preservada por componente

| Componente incluido | Estado que conserva | Lo que la release no hace |
|---|---|---|
| Gobernanza `v0.2.0` | Aprobada y normativa dentro del proyecto | No convierte el CDI-BoK en estándar externo |
| Núcleo `v0.4.0` | Aprobado con autoridad diferenciada | No cambia definiciones ni amplía su alcance |
| Caso B2B `v0.5.0-rc.1` | Candidato, instrumentado y no ejecutado | No inventa owner, acción, outcome ni efecto |
| Aprendizaje `v0.6.0-rc.1` | Guía candidata ratificada | No demuestra aprendizaje o adopción real |
| Medición `v0.7.0-rc.1` | Síntesis candidata ratificada | No valida pesos, benchmarks ni un score universal |
| Patrones `v0.8.0-rc.1` | Taxonomía candidata ratificada | No la vuelve exhaustiva, científica o eficaz por decreto |

La estabilidad corresponde al **sistema de publicación y a la línea base integrada**, no a una elevación uniforme de todos los documentos a `approved`.

## Gates de promoción

El RC se fusionó mediante el PR #10 en `bbc5c0583b28bbc3d4d2b6a2aedf28f8df336347`. Sobre ese commit aprobaron la validación post-merge `29871891960` y el despliegue `29871892015`; el portal respondió HTTP 200 y mostró la sección de patrones. El PR de promoción debe repetir preflight, build, navegador y Axe. Después del merge, el workflow estable repite esos gates y crea `v0.8.0` sobre el SHA exacto validado.

## Qué no significa “estable”

- CDI continúa siendo un dominio de práctica interdisciplinar propuesto.
- PULSE no se presenta como metodología científicamente validada ni como garantía de mejores decisiones.
- La taxonomía de patrones no es consenso académico, ley universal ni prueba de eficacia.
- El caso B2B sigue usando datos simulados y no aporta impacto observado.
- Los análisis automatizados no demuestran conformidad completa con WCAG 2.2 AA.
- `0.x` sigue indicando desarrollo fundacional; esta release no equivale a `1.0`.

## Identificación inmutable

ADR-024 autoriza el tag `v0.8.0` y el GitHub Release. Ambos se crean únicamente después del merge y de los gates estables; `v0.8.0-rc.1` permanece sin tag conforme a ADR-022.
